# Project Structure

This repository contains the WS Inference Engine for audio processing. Below is the organization of the project:

## Directory Structure

### `/audio_samples`
Contains WAV files used for testing the models. These sample files allow you to verify the functionality of the inference engine with the provided models.

### `/docs`
Documentation generated by Doxygen. This provides detailed API references and usage guides for the library.

### `/include`
Header files required for using the inference engine:
- `AudioModel.h`: Main interface for audio model processing

### `/lib/linux`
Contains the WS inference engine binary for Linux platforms. This is the core component that performs the audio inference.

### `/examples/wav_processing`
Example code demonstrating how to use the WS inference engine with WAV files. This serves as a practical guide for implementing the engine in your own applications.

### `/models`
Contains pre-trained WS models in a proprietary binary format that can be loaded by the inference engine.

## Configuration Files

- `.gitignore`: Specifies intentionally untracked files to ignore
- `CMakeLists.txt`: CMake build configuration file
- `README.md`: Main project documentation
- `setup_libs.sh`: Script to set up required Linux libraries, automatically executed during the CMake build process for Unix/Linux systems.

# Waveshaper Models

## Mic Upgrade
**Mic Upgrade** is a high-fidelity microphone modeling solution. It uses the **Waveshaper Engine** to replicate the analog circuitry of a top-tier microphone, accurately capturing its frequency response and harmonic characteristics.

## Space Helmet
**Space Helmet** is a voice modulation model that consolidates a chain of audio effects into a single processing block. A user-adjustable parameter controls the intensity of the effect—from subtle modulation to a pronounced, helmet-radio effect.

## Spectral Enhancement
**Spectral Enhancement** processes incoming signals to apply a specific mastering style. Trained on two distinct mastering profiles, it allows the user to switch between a more mid/high-focused enhancement and a low-end–oriented enhancement via a single parameter setting.
